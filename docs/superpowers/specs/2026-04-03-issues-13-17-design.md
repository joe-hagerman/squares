## Issue #13 — Payment Flow After Claiming

### Summary

After a player finishes picking squares in JoinFlow, they are directed to a dedicated payment step before the confirmation screen.

### Flow

```
INFO → PICK → PAYMENT → DONE
```

The PAYMENT step is inserted between PICK and DONE. It activates when the player taps "Done" on the PICK screen.

**Skip condition:** If the board's admin has no `cashapp_handle` and no `venmo_handle`, the PAYMENT step is skipped entirely and the flow goes directly to DONE (preserving current behavior).

### PAYMENT Step UI

- Header showing amount owed: `{mySquareIds.size} squares · $X due`
- CashApp deep-link button (via existing `PaymentMethod` component) — shown only if `admin.cashapp_handle` is set
- Venmo deep-link button (via existing `PaymentMethod` component) — shown only if `admin.venmo_handle` is set
- **"I've sent payment"** CTA — calls Supabase to set `payment_pending = true` on all the player's squares, then advances to DONE
- **"Pay later"** ghost button — skips payment marking, advances to DONE

### Implementation Notes

- `STEPS` constant gains a new value: `PAYMENT: 'payment'`
- A `markPaymentSent()` async function (mirrors the one in `PlayerView`) iterates `mySquareIds`, updates `payment_pending = true` for each square via Supabase, then calls `setStep(STEPS.DONE)`
- The "Pay later" button just calls `setStep(STEPS.DONE)` directly
- `PaymentMethod` component is already imported in `PlayerView` — import it into `JoinFlow` as well
- The `admin` state already fetches `cashapp_handle` and `venmo_handle` — no new data fetching required

## Issue #17 — Share Icon in Player and Live View Headers

### Summary

A share icon is added to the header of both `PlayerView` and `BoardView`. Tapping it opens a bottom sheet with shareable URLs and QR codes, with content that varies based on board lock status.

### ShareSheet Component

New reusable component: `src/components/ShareSheet.jsx`

**Props:**
- `boardId` — string UUID
- `isLocked` — boolean (controls which share options appear)
- `onClose` — callback to dismiss

**Content (when board is locked):**
- One section: "Watch Live" — live board URL (`/board/:boardId`) + QR code + copy button

**Content (when board is open):**
- Section 1: "Watch Live" — live board URL (`/board/:boardId`) + QR code + copy button
- Section 2: "Join Board" — join URL (`/board/:boardId/join`) + QR code + copy button

**Presentation:**
- Fixed overlay, slides up from the bottom
- Semi-transparent dark backdrop — tapping it dismisses the sheet
- X button in the sheet header to dismiss
- Matches existing dark surface design language (`var(--sq-surface)`, amber accent, IBM Plex Mono labels)
- QR codes rendered with the existing `QRCodeDisplay` component

### Placement

**PlayerView header (top-right):**
- Share icon sits alongside the existing print icon
- `boardId` is already available in component state

**BoardView header (top-right):**
- Share icon added to the right side of the board header
- `boardId` is the URL param already in scope

### Share URLs

| Condition | URL | Label |
|---|---|---|
| Always | `${window.location.origin}/board/${boardId}` | Watch Live |
| Board is open | `${window.location.origin}/board/${boardId}/join` | Join Board |
